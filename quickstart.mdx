## Installation

<Steps>
  <Step title="Install Qwen CLI globally">
    ```bash
    npm install -g @qwen-code/qwen-code
    ```
    
    Verify the installation:
    ```bash
    qwen --version
    ```
  </Step>
  
  <Step title="Get your OpenRouter API key">
    1. Visit [OpenRouter.ai](https://openrouter.ai) and create an account
    2. Navigate to the **API Keys** section
    3. Generate a new API key (starts with `sk-or-v1-`)
    4. Copy and save this key securely
    
    <Warning>
    Keep your API key secure and never commit it to version control. Store it in environment variables or secure configuration files.
    </Warning>
  </Step>
  
  <Step title="Configure authentication">
    Set up your API key with Qwen CLI:
    
    <Tabs>
      <Tab title="Interactive Setup">
        ```bash
        qwen auth
        ```
        
        Follow the prompts to enter:
        - **API Key**: Your OpenRouter API key
        - **Base URL**: `https://openrouter.ai/api/v1`
        - **Default Model**: Choose from available models
        
      </Tab>
      
      <Tab title="Command Line">
        ```bash
        qwen auth --key sk-or-v1-your-api-key \
                  --base-url https://openrouter.ai/api/v1 \
                  --model qwen/qwen-3-coder
        ```
      </Tab>
      
      <Tab title="Environment Variables">
        ```bash
        export OPENROUTER_API_KEY="sk-or-v1-your-api-key"
        export OPENAI_BASE_URL="https://openrouter.ai/api/v1"
        export QWEN_DEFAULT_MODEL="qwen/qwen-3-coder"
        ```
      </Tab>
    </Tabs>
    
    <Note>
    For a more detailed authentication guide, visit our [Authentication setup page](/authentication).
    </Note>
  </Step>
  
  <Step title="Test your setup">
    Verify everything is working:
    ```bash
    qwen "Write a hello world function in Python"
    ```
    
    You should see a Python function generated by the AI model.
  </Step>
</Steps>

## Your First Commands

Now that you're set up, try these essential commands:

### Code Generation

<CodeGroup>
```bash Generate a function
qwen "Create a JavaScript function to validate email addresses"
```

```bash Generate a component
qwen "Build a React component for a user profile card with TypeScript"
```

```bash Generate an API
qwen "Create a Express.js REST API for a todo application with CRUD operations"
```
</CodeGroup>

### Code Review and Analysis

<CodeGroup>
```bash Review code
qwen review ./src/components/Header.tsx
```

```bash Analyze performance
qwen analyze "Find performance bottlenecks in this React app"
```

```bash Debug issues
qwen debug "ReferenceError: useState is not defined"
```
</CodeGroup>

### Documentation and Explanations

<CodeGroup>
```bash Explain code
qwen explain ./utils/database.js
```

```bash Generate docs
qwen docs --input ./api --format markdown
```

```bash Create README
qwen readme --project-name "My Awesome App"
```
</CodeGroup>

## Model Selection

Choose the right model for your use case:

<Tabs>
  <Tab title="Qwen 3 Coder">
    **Best for**: Code generation, debugging, refactoring
    ```bash
    qwen --model qwen/qwen-3-coder "Optimize this SQL query"
    ```
    
    **Strengths**:
    - Excellent code quality
    - Fast response times  
    - Cost-effective
    - Supports 40+ programming languages
  </Tab>
  
  <Tab title="Kimi K2">
    **Best for**: Autonomous workflows, complex applications
    ```bash
    qwen --model moonshotai/kimi-k2 "Build a complete e-commerce platform"
    ```
    
    **Strengths**:
    - Agentic capabilities
    - Large context window (130k tokens)
    - Excellent at multi-step tasks
    - Great for full application generation
  </Tab>
  
  <Tab title="Grok 4">

    **Strengths**:
    - Advanced reasoning capabilities
    - Excellent system design skills
    - Great for complex problem solving
    - Strong analytical abilities
  </Tab>
</Tabs>

## Configuration Options

### Default Model

Set your preferred default model:
```bash
qwen config set model qwen/qwen-3-coder
```

### Output Format

Configure how responses are displayed:
```bash
qwen config set output-format markdown  # or json, plain
```

### Context Settings

Adjust context and response length:
```bash
qwen config set max-tokens 4096
qwen config set temperature 0.1
```

### Project Context

Set up project-specific configurations:
```bash
# In your project directory
qwen init

# This creates a .qwen.json file with project settings
```

<CodeGroup>
```json .qwen.json
{
  "model": "qwen/qwen-3-coder",
  "context_files": [
    "package.json",
    "README.md",
    "src/**/*.ts"
  ],
  "ignore_patterns": [
    "node_modules/**",
    "dist/**",
    "*.log"
  ],
  "custom_instructions": "Always use TypeScript and follow our coding standards"
}
```
</CodeGroup>

## Usage Patterns

### Interactive Mode

Start an interactive session:
```bash
qwen interactive
```

This opens a chat-like interface where you can have ongoing conversations with the AI.

### File Processing

Work with existing files:
```bash
# Analyze a file
qwen file analyze ./src/utils.js

# Refactor a file
qwen file refactor ./components/Dashboard.tsx --style functional

# Add tests
qwen file test ./services/api.js --framework jest
```

### Batch Operations

Process multiple files:
```bash
# Review all TypeScript files
qwen batch review --pattern "**/*.ts"

# Add documentation
qwen batch document --input ./src --output ./docs
```
